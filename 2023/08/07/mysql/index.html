数据库

数据库就是存储数据库的仓库，数据库是一个以某种有结构的有顺序存储的数据集合

数据库相关概念

名称 全称 简称
数据库 存储数据的仓库，数据是有组织的进行存储 DataBase (DB)
数据库管理系统 操纵和管理数据库的大型软件 DataBase Management System (DBMS)
SQL 操作关系型数据库的编程语言，定义了一套操作关系型数据库统一标准 Structured Query Languge (SQL)

关系型数据库

安全系数高，容易理解，比较浪费磁盘空间

  • Oracle：大型的收费数据库
  • MySQL：开源免费的中小型数据库
  • SQL Server：Microsoft公司推出的收费的中兴数据库
  • PostgreSQL：开源免费的中小型数据库
  • DB2：IBM公司的大型收费数据库产品
  • SQLLite：嵌入式的卫微型据库
  • MariaDB：开源免费的中小型数据库

非关系型数据库

安全系数低，效率高

典型代表有：NoSQL、BigTable (Google)、Cassandra、MongoDB、CouchDB

还包括键值数据库：Apache、Cassandra (Facebook)、LevelDB (Google)

区别

  • 关系型数据库：实例->数据库->表->记录行、数据字段
  • 非关系型数据库：实例->数据库->集合->键值对

数据库端口号

关系型数据库 MySQL（3306）、Oracle（1521）、Sql Server（1433）、DB2（5000）、PostgreSQL（5432）、DM达梦（5236）
非关系型数据库 Redis（6379）、Memcached（11211）、MongoDB（27017）

SQL基本语句

  • create database 库名 //创建数据库
  • create table 表名 //创建新表

  • drop database 库名 //删除数据库

  • drop table 表名 //删除表

  • select 列名 from 表名 //查看表中某一列的信息

  • desc 表名; //查询表中所有字段信息

  • select distinct 列名 from 表名 //查看表中某一列的信息并过滤重复的值

  • select 列名 from 表名 where 列 运算符 值 //有条件的筛选数据

AND 和 OR

​ AND 和 OR 可在 WHERE 子语句中把两个或多个条件结合起来

​ 条件一和条件二同时成立，则 AND 语句显示一条记录

​ 条件一和条件二至少有一个成立，则 OR 语句显示一条记录

  • select 列名 from 表名 where 条件一 and 条件二； //查询同时满足两个条件的数据

  • select 列名 from 表名 where 条件一 or 条件二； //查询至少满足其中一条条件的数据
  • select 列名 from 表名 order by 列名; //顺序查询
  • select 列名 from 表名 order by 列名 DESC; //逆序查询

  • insert into 表名 values (值1,值2,......); //像表格插入新的行
  • update 表名 set 列名 = 新值 where 列名 = 某值; //修改表中数据
  • delete from 表名 where 列名 = 值; //删除表中的数据

SQL高级语句

  • select 列名 from 表名 where 列名 like 值; //搜索指定的值

  • select 列名 from 表名 where 列名 in (值1,值2); //规定多个值
  • select 列名 from 表名 where 列名 between 值1 and 值2; //选区范围内的数据
JOIN语句

JOIN：如果表中有至少一个匹配，则返回行
LEFT JOIN：即使右表中没有匹配，也从左表返回所有的行
RIGHT JOIN：即使左表中没有匹配，也从右表返回所有的行
FULL JOIN：只要其中一个表中存在匹配，就返回行

  • select 列名1,列名2,... from 表名1 inner join 表2 on 表1列中的值 = 表2列中的值; //联合查询多个表中的数据

union语句

  • select 列名 from 表名1 union select 列名 from 表名2; //联合两表查询数据

  • alter table 表名 add 列名 类型; //表中添加列

  • alter table 表名 drop 列名 类型; //表中删除列

数据库结构

  • information_schema
    • schemata //所有库名
      • schema_name //库名
    • tables //所有表名
      • table_schema //表所属库名
      • table_name //表名
    • columns //所有字段名
      • table_schema //字段所属库名
      • table_name //字段所属表名
      • column_name //字段名
